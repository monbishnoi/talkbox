Talk Box Runtime

Talk to your agent.

This page tests the thin Talkbox runtime: one clean user turn goes to your agent, the agent stays the brain, and Talk Box speaks the result without inventing answers.

Experiment History

Test 1

Hume EVI bridge

Proved the adapter idea, but Hume owned too much turn-taking and voice output was inconsistent.

Decision: keep as adapter, not core.
Test 2

Provider-neutral runtime

Created the swappable STT/TTS seam, provider doctor, benchmark matrix, and stage timings.

Runtime overhead: p50 3-6ms.
Test 3

Deepgram -> agent -> Piper

Clean, measurable pipeline. Strong control, but batch flow left noticeable dead air.

Live baseline: 12.23s total.
Test 4

Realtime voice brain + agent

Realtime handled fillers, pacing, summaries, and drill-down while the backend agent stayed the source of truth.

Winner: best felt latency.

Benchmark Comparison

Architecture Measured Wait Voice Behavior Control Verdict
Test 1: Hume EVI -> Talk Box -> agent Unstable live path; output inconsistent Natural voice provider, but partial turns and stale state leaked into the agent. Low: provider owned too much of the loop. Adapter only
Test 2: Provider-neutral runtime Runtime p50 3-6ms; Piper p50 410ms; Sherpa p50 713ms Excellent instrumentation; not a final conversational experience. High: one clean completed turn, provider-swappable. Benchmark base
Test 3: Deepgram STT -> agent -> Piper TTS 12,230ms total; STT 1,508ms; agent 10,380ms; TTS 340ms Reliable but batchy. Spoken output was too literal/truncated for long agent answers. Very high: the agent is the task brain, Talk Box is deterministic. Safe baseline
Test 4: OpenAI Realtime voice brain -> agent Agent tool p50 26,475ms across 7 successful calls Best experience: graceful fillers, intelligent summaries, and follow-up drill-down. High when bounded by ask_agent: the agent owns truth, memory, and tools. Recommended

Key finding: raw backend latency and felt latency diverged. Test 4 waited longer for the agent, but felt faster because the voice front layer kept the conversation alive while the agent worked.
