Talkbox Architecture Flow

Cal reference integration

Your agent owns the brain. OpenAI Realtime owns the voice. Talkbox brings them to life.

Cal PWA and Cal Gateway are one product boundary. Talkbox is the separate voice-orchestration server. OpenAI Realtime hears, speaks, and manages the live voice interaction.

Implementation detail: Talkbox does not own the WebRTC data channel. Cal PWA owns it and sends conversation.item.create and response.create directly to OpenAI Realtime after Talkbox returns an HTTP result or instruction.

Boundary principle

Talkbox may manage conversation mechanics, but it must not invent task answers. It authors persona, pacing, preambles, narration policy, and spoken rendering instructions.

Cal owns task truth. Its memory, tools, and reasoning execute behind ask_agent. Cal PWA transports returned results directly to OpenAI Realtime.
